# Стиль кода

## Принципы: KISS · DRY · SOLID

- **KISS** — не усложняй. Каждая функция делает ровно одну вещь. Если функция не помещается на экран (50 строк) — разбей.
- **DRY** — никакого копипаста. Повторяющийся паттерн → функция или макрос.
- **SOLID** — единственная ответственность, открытость к расширению, подстановочность, разделение интерфейсов, инверсия зависимостей. В ядре это значит: модули не знают друг о друге, общаются через функции.

## Форматирование

- `cargo fmt` обязателен
- Имена: snake_case для функций/переменных, CamelCase для типов
- Константы: UPPER_SNAKE_CASE
- unsafe — только MMIO и ассемблерные вставки
- Минимум комментариев, только там где логика неочевидна

## Ограничения

- Максимум 150 строк на файл. Не влезает → разбивай на подмодули.
- Максимум 3-6 файлов на директорию. Больше → группируй в поддиректории.

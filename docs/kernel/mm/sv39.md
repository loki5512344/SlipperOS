# Sv39 Virtual Memory

RISC-V 64-bit использует Sv39: 39-битные виртуальные адреса, 3 уровня страничных таблиц.

## Virtual Address (39-bit)

```
|  VPN[2] (9) | VPN[1] (9) | VPN[0] (9) | offset (12) |
| bits 38-30  | bits 29-21 | bits 20-12 | bits 11-0   |
```

Каждый VPN — индекс в таблице из 512 entries (8 байт = 4096 байт на таблицу).

## PTE (Page Table Entry) — 64 бита

| Бит | Поле | Описание |
|---|---|---|
| 0 | V | Valid |
| 1 | R | Readable |
| 2 | W | Writable |
| 3 | X | Executable |
| 4 | U | User-accessible |
| 5 | G | Global |
| 6 | A | Accessed |
| 7 | D | Dirty |
| 8 | RSW | Reserved for SW |
| 10-53 | PPN[2:0] | Physical Page Number |
| 54-63 | Reserved | |

## Leaf vs Branch

- Если **R/W/X** все 0 → **branch** (PTE указывает на下一页 таблицу)
- Если любой R/W/X = 1 → **leaf** (PTE указывает на физическую страницу)

Любой уровень может быть leaf:
- L2 leaf → 1GiB page (PPN[2] from PTE, PPN[1]=VPN[1], PPN[0]=VPN[0])
- L1 leaf → 2MiB page (PPN[2:1] from PTE, PPN[0]=VPN[0])
- L0 leaf → 4KiB page (PPN[2:0] from PTE)

## SATP Register

```
|  MODE (4) | ASID (16) |  PPN (44)  |
| bits 63-60| bits 59-44| bits 43-0  |
```

- MODE = 8 → Sv39
- PPN = physical address of root table >> 12
- ASID = Address Space ID (TLB tagging)

## address layout (SlipperOS)

| Регион | Адрес |
|---|---|
| Kernel text | 0x80000000 (identity) |
| UART MMIO | 0x10000000 (identity) |
| CLINT | 0x02000000 (identity) |
| PLIC | 0x0C000000 (identity) |
| VirtIO | 0x10001000 (identity) |
| User stack | 0x00001000-... (per process) |
| User text | 0x10000 (per process) |

## map_page

```rust
fn map_page(root, vaddr, paddr, flags):
    for level in [2, 1]:
        if PTE is invalid:
            alloc page, set PTE = page_addr | V
        next = PTE.PPN << 12
        advance to next level
    // level 0: set leaf PTE
    PTE = (PPN << 10) | flags | V
```
